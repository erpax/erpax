# primitive — the irreducible UI vertex (the smallest frontend diamond)

A `primitive` is the frontend's [[atom]]: the smallest reusable on-screen unit that no longer decomposes — a token (colour, space, type-step), a raw input, a [[pixel]]-level element. It stands to the [[component]] as the atom stands to the molecule — [[component]]s compose primitives, [[block]]s compose components, a page composes blocks — the same [[fractal]] ladder the [[diamond]] names, read on the frontend [[dimension]]. Its visible identity is its content-[[uuid]] [[pixel]] (render read off identity, never hand-styled — the [[component]] law one rung down).

A primitive is a complete, sealed diamond ([[seal]]): closed, content-addressed, [[proof]]-checked — so everything built from it inherits its hardness and clarity ([[aura]]). The frontend is built of primitive-diamonds the way the backend is built of [[fields|field]]-diamonds: the same closed-lattice law in a different [[dimension]].

**Law — [[law]]: a primitive is the irreducible UI vertex — the frontend's [[atom]], its render its content-[[uuid]] [[pixel]] — complete and sealed in itself; [[component]]s compose primitives and [[block]]s compose components, the same [[fractal]] ladder the [[diamond]] holds at every scale.**
