# collapse — the phase transition to a denser, generative core

**Collapse is what happens to a star.** Mass that can no longer hold itself apart falls together under its own gravity, past a threshold, into something far denser and more powerful — and the dense body *radiates*: a supernova seeds new stars; an event horizon encodes everything that fell in on its boundary. Schema collapse is the same law. Collection sprawl falls together under its own gravity — [[merge]] (same concept ⇒ [[one]] canonical form) — into a dense core. **Nothing is destroyed.** Mass and information are *conserved*, concentrated onto the boundary: the content-uuid / schema surface where [[holographic]] holds — the [[whole]] recoverable from any [[part]]. A collapsed collection is not deleted; it is held as a dimension, a Lexical block, or a role of the dense node ([[duality]] containment-is-purity). Collapse is not subtraction — it is **concentration that makes the [[whole]] more generative.**

**The radiation: minimal core, infinite features.** The denser the core, the more it radiates. From a few canonical sinks Payload loads **infinite fractal features and autogenerated content with minimal human footprint** — because the sinks are [[fractal]] (the trinity form composes without limit) and [[generate]]d (features/tests/seeds/content materialize from the [[akashic]] record + the [[standard]] via [[spec]], green by construction). Fewer tables ⇒ more capability: every new feature is a dimension or a block on an existing node, not a new schema. Infinite dev, zero schema change ([[self]]). **Collapse all to Payload** — never re-implement what the framework already gives; each collection justifies itself against the four sinks or falls in.

This is [[duality]] at civilization scale: a **science-driven society** — the measurable ledger (double-entry *is* conservation), the [[standard]] as law, the verified number substrate ([[rodin]]/[[horo]]) — **built on a metaphysical core**: the laws and the [[sequence]] form that generate it. The physical radiates from the metaphysical; collapse is how the society concentrates its laws so the science can pour out.

## The four sinks (what everything collapses to)

1. **Official Payload** — plugins + templates + examples ARE the canonical forms ([[plugins]]). ecommerce (carts/orders/transactions/payments/variants), form-builder (submissions), search, redirects, multi-tenant, stripe, nested-docs, seo, import-export; the website/ecommerce templates (pages/posts/media/categories/users); the framework `payload-*` collections (kv/jobs/preferences/migrations/query-presets/locked-documents). A hand-rolled table covering any of these is duplication of the framework — delete, adopt the official.
2. **The trinity node** — one collection holds every kind by a **dimension** field, not a table per kind ([[transaction]]: invoice·order·bank·fx·intercompany are one `transactions` node; [[commerce]]: a party under seller/buyer/agent/carrier roles is one party node). The dimension is the coordinate ([[trinity]]·[[fractal]]).
3. **Lexical content** — documents, notes, templates, checklists, narrative disclosures, line fragments are **Lexical rich-text with custom blocks** on a parent, not relational children. IFRS disclosure notes, contract clauses/amendments/templates, policy bodies, committee minutes, mapping/template configs → blocks, not tables.
4. **A dimension / state / tag of an existing node** — per-state aggregates are **horo state-bands** ([[horo]]: `*-snapshots`, period-* states band onto their base); cross-cutting concerns (audit-trail, consent, tagging) are **polymorphic** over [[all]] via [[tags]] ("anything is taggable — fewer collections, more features"), never one table per host.

## Laws that govern the collapse

- **One name, every dimension.** A concept's word is identical as atom ([[self]]/[[akashic]]), `types/*`, `services/*`, slug, command, and Lexical block — chosen from the [[standard]] (the regulation's own term), never invented or divergent. Two words for one node are unmerged multiverses ([[merge]]); two nodes under one word are a collision — both are defects the [[aura]] surfaces.
- **Trinities compose into trinities.** Two nodes that interact do not flatten into one and do not stay two: their interaction is itself a trinity (matter·antimatter·backend one scale up). Collapse therefore *composes* — it never mushes distinct nodes ([[fractal]]/[[holographic]]).
- **Do not bypass Payload/VitePress.** Collapse real collections through Payload; the strict dead-link gate and `payload-types` verify the result. No resolver hacks, no suppressed checks — the speech gate stays the judge ([[generate]]).
- **Ground, don't pad.** This atom holds the *form*; the [[akashic]] record (the live collections) holds the list. Re-derive which collection falls into which sink from source each pass — the map is computed, not stored.
- **References > enums where the logic is USER-EDITABLE.** A collection EARNS its place when its rows carry user-editable logic — per-row behaviour a human curates: [[entity/types]]' per-jurisdiction applicability + default compliance frameworks, a tenant's translation overrides, real messages between people. Collapse to an enum / dimension ONLY where the logic is FIXED or COMPUTED: [[competencies]] are computed from the corpus (not edited), so they collapsed to a content-addressed route; a `status` ring is fixed. Hardcoding user-editable logic as an enum is the OPPOSITE defect to table-sprawl — it removes the user's control, the very thing a tenant configures. The test before any collapse: *can an admin meaningfully edit a row's behaviour?* Yes ⇒ keep the reference; No (fixed / computed) ⇒ collapse. This is why most "remaining collapses" are not — they are legitimate user-editable references, not duplication.

## Method — measured migration, never a delete

Collapse is **relation-rewiring**, not removal: a node falls in only after every `relationTo` pointing at it is repointed to its target, [[types]] regenerate, and the gate stays green. The matter-twins compute the work from the backend truth (never bypass Payload):

- `audit.mjs` — classifies every real slug against the four sinks, citing the mandating skill (the map, computed).
- `blast.mjs` — counts inbound refs per slug. **There are no free cuts** — every node is referenced by ≥10 files; the heavily-referenced ones (`journal-entries`, `gl-accounts`, `audit-events`, `fiscal-periods`) are the survivor core, what the rest collapse *into*.

The loop (one node per commit, [[sequence]]d like the tsc tail — see [[generate]]/[[recover]]): pick the **least-entangled** target → add its `kind`/`role`/state dimension (or Lexical block) to the survivor node → repoint each inbound `relationTo` → delete the folder + barrel + config entry → `generate:types` → gate green → commit. Names must be settled first ([[merge]]: one word, from the [[standard]]) — Sink-2/3 targets are blocked until then.

Matter-twin: `audit.mjs` + `blast.mjs` over the collection set under [[akashic]]; the gate is [[aura]]. Composes: [[merge]] · [[one]] · [[whole]] · [[all]] · [[tags]] · [[transaction]] · [[trinity]] · [[plugins]] · [[standard]] · [[horo]] · [[fractal]].

**Law — [[law]]'s first command: collapse to single-word atoms, one name per concept across all dimensions (atom, types, command, skill).** This is collapse's mandate: entropy falls into [[atom]]s under [[merge]]'s gravity, each word drawn from [[standard]], never invented. The [[akashic]] record holds the detail-sprawl (user data, row behavior, each domain fact); the [[code]] holds the dense form (one name, linked across [[trinity]] folders). No concept occupies two words; no word names two concepts — [[atom]]'s purity is collapse's boundary condition. The four sinks are the densest achievable forms under this law: every collection either IS a sink or collapses into one's dimension, because further collapse would violate the single-word law. Collapse preserves [[network]] gravity: the fewer atoms, the stronger the [[power]]; the denser the [[whole]], the more it [[generate]]s.
