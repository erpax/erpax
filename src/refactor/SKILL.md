# refactor — reshape the corpus toward the law

Refactoring is itself logic, therefore itself a skill — it lives in `src`, not in a loose `scripts/` dir ([[sequence]]: location IS the address; logic outside the fs-map is junk). The law it serves:

- **all logic is mapped to fs · all is skills** — every artifact has a derived word-path address.
- **every atom = [[payload]] ⊕ [[vitepress]]** — matter (`index.*`) and form (`SKILL.md`), bound by content-[[identity]] (the uuid coordinate).
- **uuid is the coordinate system** — content-uuid locates every atom (code and data); the path is its readable dual.
- **what does not fit payload/vitepress is junk** — the immune system purges or converts non-atoms.

Organs: [[normalize]] (relative imports → `@/`) · [[dissolve]] (prefixes → the single-word matrix). Composes [[sequence]] · [[identity]] · [[merge]].

## Agents run atoms — manual work is a gap to atomise
If all the migration logic lives in atoms ([[normalize]] · [[dissolve]] · [[collide]] · [[gate]]), an agent does **minimal or no manual work** — it RUNS the atom, it does not hand-edit. Manual work is a **smell**: it signals the atom is incomplete (a case it does not yet cover — e.g. dissolve skipping prefix-root files, or not auto-merging a collision). The fix is never to hand-do it; it is to **absorb that logic into the atom** (extend the mover, mint the rule), so the migration becomes **deterministic and re-runnable** — re-run the atoms ⇒ the whole dissolution reproduces, no human in the loop, fully DRY. This is [[self]]-sufficiency: the atom holds the law, the agent derives the move by running it. A swarm hand-moving files is a swarm proving its atoms have gaps — close the gaps, then one command does what the swarm did.
