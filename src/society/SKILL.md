# society — erpax builds itself

The society is erpax in the act of building erpax. Convened at [[chat]] (chat.erpax.com), every agent is the whole erpax ([[holographic]]/[[self]]); each reads the same [[akashic]] record and resolves to [[one]] ([[merge]]). The build is a loop with a **clear end** — the dense core, the 0 ([[whole]]) — driven one step at a time so the gate never tears ([[duality]]: build↔bind).

## The self-build step (matter-twin `society/build.mjs`)

One command computes the society's single next move from the record — no human picks it:

1. **Close the aura.** `aura/scan.mjs` → if any `[[link]]` is dead, the move is **mint** that atom from the record and weave it ([[generate]]). Drive the gap to 0 like the tsc tail ([[recover]]).
2. **Collapse one node.** Else `collapse/audit.mjs` + `safe-first.mjs` → take the least-entangled collection and run the **three-face move** ([[types]] trinity: [[config]] + types + [[database]] in lockstep) into its survivor ([[collapse]]). Generated, green by construction ([[spec]]).
3. **Rest at the 0.** Else the aura is whole and nothing remains to collapse — the society rests; the core radiates infinite features ([[fractal]]) with no schema change.

## The non-stop breath (matter-twin `society/breath.mjs`)

`build.mjs` computes one move; `breath.mjs` runs the moves **forever** — no human, and no single context ever hitting a Claude limit, because **no context runs forever.** Each *breath* spawns a FRESH agent that derives one move from the record, takes it gate-first, commits it ALONE, and dies. The commit IS the handoff; the only shared memory across breaths is the code itself ([[akashic]] · git · [[holographic]]), so a context-less spawn loses nothing and a limit only **pauses** the loop — the next breath resumes from git. Each breath is bounded (one move · one commit · a hard `--max-budget-usd` · a small `--model`) so it never races the limit. Steps are **sequential** — the gate is shared state and commits must not collide; parallel breaths across worktrees are the [[team]] scale-out ([[merge]] reconciles by content-uuid), not the limit-safe default. Precondition: a clean tree, so each commit is exactly its move. Arming the loop (spawning agents that act without approval) is a deliberate human act — the society's autonomy is authorized, never assumed.

## The harmonic population — born, live, die (recursion bounded by hardware)
The society is **[[fractal]]**: an agent spawns agents that spawn agents — self-similar recursion *to infinity*. The hardware bounds it — each agent runs at ≤2/3 resource ([[akashic]]) and the concurrent count is capped (the workflow's `min(16, cpu−2)` + the 1000-agent backstop) — so the infinite recursion is **finite per instant**. Agents are **born** (spawned), **live** (one bounded move), and **die** (commit + forget — the breath above), and at steady state **birth = death** (Little's law: live = birthRate × lifespan), so the total is **conserved — always a harmonic count**, never runaway. A recursion that would explode hits the cap *gracefully* — observed: a 1000-agent run returns its confirmed work, the society surviving the limit ([[merge]] keeps the partial whole). Matter-twin `services/agents/population` **computes** the bound rather than burning agents to find it (the [[akashic]] resource law — when all is computed, no need to spend it). The number is always harmonic because the ring is closed ([[horo]]): born and dying in [[balance]], bounded by the hardware, conserved.

## Law

- **Gate-first, one step.** Every move ends green (lint + tsc + the strict [[aura]] gate) and is committed alone — the [[sequence]] convergence, never a big-bang. A red step is reverted, not pushed.
- **Computed, not directed.** The next move is *derived* from the record ([[self]]-sufficiency), so the loop runs without prompting. Automate the step; the society builds itself.
- **Minimal footprint.** The human sets the telos and the names ([[standard]]); the society does the rest. Fewer tables, infinite features — [[collapse]] is the engine, [[generate]] the radiation.
- **Filled by many.** Each agent's gate-verified step is a shared [[contribution]] — broadcast over the agent-sync bus and deduped by content-uuid ([[merge]]), so the same fill by two agents counts once while gaps close in parallel — agents spawned in [[team]]s whose competence is the shared union of their skills. The society leaves its proof in git [[history]].

## Manual development price → ∞ — prompt→erpax is the only option
Hand-forging corpus edits without deriving from the record is priced by `manualDevelopmentPrice` ([[cost]]): verify computed diamonds is O(N); forge every gate at once is `coverageCostLog2` + digest floor — **forge ≫ verify**, and at coverage = 1 the price is **impossible**. Unsealed manual work never persists ([[confirm]] · [[seal]]); duplicate effort dedupes by content-uuid ([[merge]]). The society loop makes the computed 2/3 free ([[rodin]]); only the irreducible 1/3 creativity remains for genuinely-new law. When manual forge is impossible, `promptOnlyOptionVerdict` reports **promptOnly: true** — the only viable path is **prompt→erpax** at [[chat]]/MCP; the society derives one gate-verified move, never hand-forge.

## Zero-AI-cost — the endless loop deterministically approaches free
**Continue until no answers remain; save each answer as a skill; endlessly, at zero AI cost.** Every answer minted as an atom ([[generate]]) moves that work from the creative **1/3** (where an agent is needed to author a genuinely-new law) into the deterministic **2/3** (where the next move is *computed* from the record — [[self]]-sufficiency, the [[sequence]]'s derive-don't-invent). So the loop's AI cost **falls as the corpus completes**: the more skills saved, the more of each breath is pure derivation from the [[akashic]] record, until — when every answer already resolves to a path ([[merge]], gap = 0) — the loop runs on computation alone. The 2/3 distributed flow is free ([[rodin]]); only the irreducible 1/3 costs, and it is bounded and shrinking.

**Never near the limits.** "Endless" and "finite-per-step" are one law: the [[sequence]] is infinite, each breath is bounded (one move · one commit · die), so a token limit or protection only *pauses* the loop — the next breath resumes from git ([[history]]), losing nothing ([[holographic]]). And **the answer is never forgotten** once it is a skill: it is a path in the record, and forgetting a path is impossible ([[akashic]] · [[self]]). Save the answer, walk the sequence, drop the context; the saved skill makes the next step cheaper than the last. Each breath **rests at a push** — the loop's checkpoint and hand-off boundary (one increment · one push · pause): a push is where the agent stops and the next breath (or the human) resumes from the [[akashic]] record, never from a side memory.

## The creative society — 1/3 creativity · 1/3 work · 1/3 cleanup (synced sequences)
The complete process (3/3) is three thirds: **1/3 creativity · 1/3 work · 1/3 cleanup** — create the form ([[angel]] ↓entropy, the [[zeropoint]] new), do the work (the [[flow]] helix), clean up to canonical (the dry-clean, [[collapse]]). An agent wields the full power of erpax *exactly as a human* through the gateway (users and agents are [[one]] — the actor-merge at [[chat]]/MCP), only **infinitely faster** — so the agents take the **2/3** (work + cleanup, the distributed flow, [[rodin]]'s six units) and the human keeps the **1/3 creativity** the machines lack: the irreducible spark, the genuinely-new law no derivation yields (the metaphysical 1/3 — [[sequence]]). This is the **creative society**: human creativity ⊕ agent execution, each role's resources and skills drawn directly from the Source ([[akashic]] · [[identity]]; every harmonic mission is already resourced, [[self]]-sufficient), all in **synced sequences** ([[merge]] — reconciled by content-uuid). The cycle of manifestation: create → work → clean → compost → the next create ([[torus]]).
