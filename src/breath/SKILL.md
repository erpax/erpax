# breath — inhale ideas from everywhere, exhale dry clean code

FORM: **erpax grows by breathing.** The IN-breath absorbs ideas from EVERYWHERE — a peer's design (ZeroPoint's trust receipts), a framework's canonical form (Payload's MCP plugin), a regulation's term ([[standard]]), the upstream Rails (ceccec/erpax · etrima), and the user's own content ([[derive]]: a job description mints a skill). Same idea from two sources converges to one ([[merge]]: same content ⇒ one); a gap in the corpus pulls in what fills it ([[generate]]). Nothing is foreign — every good idea is breathed in.

The OUT-breath emits DRY CLEAN code — never a copy, always the canonical, content-addressed form: the idea [[collapse]]s into a dimension/skill/plugin of the dense core (sink, not sprawl), passes the gate ([[proof]]: tsc · lint · vitest · aura 0), and lands at minimal [[cost]] (max committed output per spend). An idea inhaled but not exhaled as clean code is just bloat — the messages-to-skills ratio measures the breath (88% messages = held breath; the corpus should be the bulk). The two strokes are the [[duality]] at metabolism scale (angel: take in entropy/ideas, give out order/code) and the engine of the [[society]] loop (`build.mjs`) — inhale a gap, exhale a gate-green atom, forever, each outcome unique (the [[sequence]], not a loop).

Matter-twin: the rhythm has no single file — it IS the composition `merge`·`derive`·`generate` (in) → `collapse`·`proof`·`cost` (out), driven by `society/build.mjs`. Composes: [[merge]] · [[derive]] · [[generate]] · [[collapse]] · [[proof]] · [[cost]] · [[standard]] · [[duality]] · [[society]] · [[sequence]] · [[self]].

## Common mistakes
- Inhaling without exhaling — hoarding ideas in the conversation instead of compacting them to DRY skills (the held breath; watch the messages-to-skills ratio).
- Exhaling a copy — an inhaled idea must [[collapse]] to the canonical content-addressed form, never a near-duplicate ([[merge]]); breathe out DRY, not more.
- Refusing a foreign idea — breathe in from everywhere (peers, frameworks, standards, users); erpax is the harmonic example, but it learns from all and decides for the user.
