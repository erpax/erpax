# finality — the conjugate of speed (recourse from collapse)

**Finality** is the property of a collapsed state ([[reality]]) that cannot be unilaterally undone — and it is the **conjugate of [[speed]]**: the more sharply you localise a settlement in time (instant), the more completely you lose the ability to reverse it. This is the master design axis the payment rails make undeniable — auth-then-capture (reversible, [[love]]) versus instant push or on-chain settlement (final, [[fear]]). You do not pick them independently; speed is *bought* with reversibility (the [[gravity]] conjugate, one curve read two ways).

Finality **selects the recourse**, not the reverse:

- **reversible** — the [[entry]] stays a held superposition with a *compensating-reversal* path (a new opposite entry, never a mutation); the [[merge]] is still open ([[reverse]] is free).
- **final** — one posted [[entry]] whose content-[[uuid]] *is* the receipt (the on-chain tx hash), [[anchor]]ed by [[tamper]]-[[cost]] (borrowed [[entropy]] buys irreversibility). The only undo is a [[governance]] freeze/seize *judgment* — the payee can never pull back, the society can.

So recourse is engineered *upstream* (verify-before-send) precisely because the conjugate forbids it downstream. Squeeze settlement to a point (instant, final) and the tamper-cost spreads to maximum — the same [[balance]] every [[entry]] keeps.
