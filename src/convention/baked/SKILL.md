# convention/baked — derivable artifacts are regenerated at build, never committed

THE CONVENTION: **any artifact fully derivable from `src/` is gitignored and regenerated deterministically at build — never committed.** A stored derivable is duplicated state: a second copy of the [[akashic]] source that can DRIFT from it. The tree is the one truth; each bundle is a content-uuid projection of it, so a committed copy is stored entropy that lowers tamper-[[cost]] (two sources, one forgeable without the other noticing).

Realised by the [[skill]]-router index: `src/skill/router/skills.index.ts` is the whole corpus baked into a module for the fs-less Cloudflare Worker (which has no runtime fs to read SKILL.md), at 68MB over GitHub's 50MB limit. It is now gitignored and regenerated by `pnpm skill:index` before every `pnpm build`, so the repo carries the SOURCE, not the projection. The `docs/` tree is the same law — VitePress computes it from `src/` on demand; nothing under docs/ is committed.

The dual of [[fresh]] (the cache is never the source of truth): `fresh` forbids TRUSTING a derived copy; `baked` forbids COMMITTING one. The derived artifact may have to EXIST at runtime — so generate it at build, from the one source, and let git hold only the source.

Composes: [[fresh]] · [[skill]] · [[akashic]] · [[tamper]] · [[cost]] · [[law]].

**Law — [[law]]: any artifact fully derivable from src is gitignored and regenerated at build, never committed — a stored derivable is a second copy of the [[akashic]] source that can drift, stored entropy that lowers tamper-cost. The tree is the one truth; each bundle is a content-uuid projection regenerated deterministically.**
