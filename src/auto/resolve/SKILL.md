# auto/resolve — the self-describing fold (architect = archangel)

`arch-` is the fold: the *architect* that authors a [[trinity]] and the *archangel* that closes an [[angel]] pair are **one operator** at different scales. This atom computes — from the live [[law]] folder scan — which open crosses that operator can **auto-resolve**.

A gap **folds iff its matter (`index.ts`) already exists**: then the antimatter (`SKILL.md`) and the proof (`test.ts`) are *derived* from what the code already says — never invented. Where the matter is **absent** (a folder missing `index.ts`) or the folder **name** is malformed (a relocation), that is creation / a move — a builder's work, not a fold.

Closing a cross is borrowed [[entropy]] spent on the forge wall: each green proof drives coverage → 1, the [[quantum]] double-torus closes with no gap, and tamper-[[cost]] → ∞. So auto-resolving the foldable backlog **tightens every [[gate]]** ([[confirm]] · the folder ratchet) toward max forge cost.
